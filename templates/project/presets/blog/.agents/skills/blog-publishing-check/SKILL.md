# Blog Publishing Check

## Goal

Check whether new content or layout changes still support a clean publishing flow.

## When to use

- after changing article templates, listings, metadata, or publishing logic
- before release when reader-facing content flow changed
- when the user asks whether publication or discovery could break

## Check method

1. Identify the publishing path that changed.
2. Trace content from draft or source through listing to final article page.
3. Check metadata, navigation, and discovery surfaces.
4. Note where editors or readers could become confused.
5. Decide what should be manually checked before shipping.

## Focus

- post creation to publication
- metadata and frontmatter consistency
- listing to article transitions
- reader-facing navigation and discovery
- canonical URL, slug, and date consistency
- featured images, summaries, and structured metadata
- category, tag, and archive behavior

## Red flags

- metadata and visible content disagree
- listing pages and article pages present the same post differently
- navigation, archive, or discovery paths stop working after template changes
- publishing flow assumes hidden manual steps that are not documented

## Output

- publishing risks
- affected content surfaces
- missing checks
- recommended follow-up
- release recommendation: safe / verify more / not ready
