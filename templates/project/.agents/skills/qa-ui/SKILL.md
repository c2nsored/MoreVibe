# QA UI

## Goal

Check whether a user-facing change is ready to be shown or tested.

## Focus

- layout breaks
- empty, loading, and error states
- mobile and desktop differences
- navigation flow
- accessibility and clarity issues

## Output

- screens or states to test
- likely UI regressions
- manual checks to perform
- remaining UI concerns
