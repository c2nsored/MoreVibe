# Memory Model

MoreVibe treats project memory as layered, not flat.

## `sources/`

Use this for:

- raw research
- external references
- copied logs
- exploratory notes

This layer is useful, but not automatically authoritative.

## `canon/`

Use this for:

- project overview
- architecture
- tasks
- handoff
- operational truth

This is the main source of truth for the project.

## `wiki/`

Use this for:

- current working state
- session summaries
- fast resume notes
- generated memory support

This helps continuity, but should not override canon.

## `schema/`

Use this for:

- startup order
- workflow map
- skill routing
- delegation model

This defines how the harness behaves.

## Why This Matters

Without these boundaries, long projects tend to confuse:

- evidence with decisions
- recent chat with durable truth
- temporary progress notes with handoff-ready state
