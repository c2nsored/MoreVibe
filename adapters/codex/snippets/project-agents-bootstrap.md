## MoreVibe Bootstrap

This project uses MoreVibe as a project-local harness.

- Keep the root `AGENTS.md` as the main public entrypoint.
- Treat `.morevibe/` as the MoreVibe-managed namespace.
- Treat `.morevibe/canon/` as authoritative over `.morevibe/wiki/`.
- At session start, read `.morevibe/schema/SESSION_BOOTSTRAP.md` and `.morevibe/schema/PROJECT_SKILLS.md` before broad rediscovery.
- Use `.morevibe/wiki/state.md` and `.morevibe/wiki/index.md` as the first internal navigation points when relevant.
- Use the actual project skill chain and delegation model recorded in `.morevibe/schema/PROJECT_SKILLS.md`.
- Route non-trivial work through the detected project-native workflow before implementation.
